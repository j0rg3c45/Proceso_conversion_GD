# Proyecto: Caracterización y Transformación de Datos Geoespaciales

## Estándares de Codificación

- **Estilo de Salida:** Mantener un tono profesional y directo, propio de un ingeniero de software senior.
- **Sin Iconos:** Queda estrictamente prohibido el uso de emojis, iconos o emoticonos en el código fuente, comentarios, cadenas de texto o impresiones por consola. Toda la comunicación por consola debe ser en texto plano.
- **Interacción:** Los scripts deben permitir la selección interactiva de archivos (individual, múltiple por comas, rangos o masiva). Se debe usar el número **0** como opción para procesar todos los archivos encontrados.
- **Manejo de Datos:** Priorizar la limpieza de duplicados y la estandarización de sistemas de coordenadas (EPSG:4326 para GeoJSON).
